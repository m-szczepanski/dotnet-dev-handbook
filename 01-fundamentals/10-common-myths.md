# 10. Common Myths About .NET

## 1. What problem does this solve?

Dispelling common myths about .NET helps developers build confidence in the platform and make informed decisions when choosing tools or technologies.

## 2. Core concept

Many misconceptions about .NET exist, but they often stem from outdated information or misunderstandings. Understanding these myths can help you avoid pitfalls and leverage .NET’s full potential.

## 3. Common myths debunked

- **Myth 1: “.NET is slow”**  
  Modern .NET implementations are highly optimized, especially with features like JIT compilation and garbage collection improvements.
- **Myth 2: “.NET is Windows-only”**  
  .NET supports multiple platforms, including Linux and macOS, thanks to .NET Core and .NET 5+.
- **Myth 3: “You must use Visual Studio”**  
  While Visual Studio is a powerful IDE, you can develop .NET apps using other tools like VS Code or even the command line with `dotnet`.
- **Myth 4: “.NET is dying”**  
  .NET continues to evolve rapidly, with active development and strong community support.

## 4. Practical examples

Here’s how you can debunk these myths:

```bash
# Example: Running a .NET app on Linux (debunking "Windows-only" myth)
$ dotnet run
```

Explanation:  

- This shows that .NET apps can run seamlessly on non-Windows platforms.
